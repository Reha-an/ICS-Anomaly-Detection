# Industrial Control System Malware Detection via Sensor Data Anomalies

Hybrid anomaly detection pipeline for ICS/SWaT sensor data using:
- **SARIMA** for time-series residual anomaly scoring
- **LSTM Autoencoder** for sequence reconstruction anomaly scoring
- **OR fusion** to combine detector alerts

The project trains on SWaT data from Kaggle and produces alert logs, detection results, and metrics.

## Project Contents

- `uni` - main Python pipeline script (train/simulate/dashboard modes)
- `Copy_of_Untitled21.ipynb` - notebook version of the workflow
- `ics_swat_outputs/` - generated output artifacts
  - `detection_results.csv`
  - `alerts.csv`
  - `metrics.json`
- `confusion_matrix.png`, `detection_visualization.png`, `metrics_summary.png` - result visualizations

## How It Works

1. Load SWaT dataset from Kaggle (`vishala28/swat-dataset-secure-water-treatment-system`)
2. Infer timestamp, label, and numeric sensor columns
3. Preprocess sensor data (coerce numeric, interpolate, scale, PCA)
4. Train SARIMA on a reduced principal component stream
5. Train LSTM autoencoder on PCA sequences
6. Detect anomalies with each model and fuse with OR logic
7. Save results and (if labels are available) compute metrics

## Requirements

- Python 3.9+ (3.10 or 3.11 recommended)
- Kaggle credentials configured for dataset access

Install dependencies:

```bash
pip install "numpy<2" pandas scikit-learn statsmodels tensorflow joblib matplotlib kagglehub streamlit
```

## Run the Pipeline

The main script is currently named `uni`. You can run it directly as Python:

```bash
python uni --mode train
```

### Modes

- Train models and generate batch results:
  ```bash
  python uni --mode train
  ```
- Simulate real-time detection:
  ```bash
  python uni --mode simulate --simulate_steps 20000 --sleep 0.0
  ```
- Launch Streamlit dashboard:
  ```bash
  python uni --mode dashboard
  ```

## Output Files

After training/simulation, outputs are written to `ics_swat_outputs/`:

- `detection_results.csv` - row-level scores and combined alerts
- `alerts.csv` - subset of rows flagged by fusion logic
- `metrics.json` - classification metrics (if labels exist)
- `realtime_log.csv` - generated by simulation mode

Model artifacts are saved to `ics_swat_models/` during training.

## Notes

- If Kaggle download fails, verify your Kaggle API setup and credentials.
- If LSTM training raises "Not enough sequences", reduce sequence length or increase `MAX_ROWS`.
- The script includes performance-oriented settings for faster SARIMA fitting (downsampling, capped train length).
